▶ ℹ️ KEY PAIR GENERATION - GENERACIÓN DE PAR DE CLAVES (click para ver ayuda)
Esta herramienta genera un par de claves asimétricas (clave privada + clave pública derivada) usando openssl genpkey.

📝 Datos a Ingresar:
• Key Type: Algoritmo criptográfico a usar
  • RSA: Algoritmo clásico, ampliamente soportado, basado en factorización de números primos
  • Elliptic Curves (EC): Curvas elípticas, más eficiente, claves más pequeñas con igual seguridad
• Key Length / Security Level: Tamaño de la clave en bits
  • 2048 bits: Seguridad estándar para la mayoría de aplicaciones (mínimo recomendado)
  • 3072 bits: Seguridad alta, recomendado para datos sensibles
  • 4096 bits: Seguridad muy alta, usado en infraestructuras críticas (más lento)
  • ⚠️ A mayor tamaño, mayor seguridad pero menor rendimiento
• Output File Name: Nombre sugerido para guardar la clave (sin extensión)
• Encrypt Private Key: Opción de proteger la clave privada con contraseña
  • Si activas esta opción, la clave se cifrará con AES/3DES
  • Requerirá passphrase cada vez que uses la clave privada
  • Recomendado: Siempre encripta claves privadas en producción

🔐 ¿Para qué se usa?
• Identidad Digital: La clave privada es tu "identidad" en criptografía de clave pública
• Firma Digital: Firmar documentos, código, emails (nadie más puede falsificar tu firma)
• Certificados SSL/TLS: Base para generar CSR (Certificate Signing Request) y obtener certificados
• SSH Authentication: Autenticación sin contraseña a servidores remotos
• Cifrado Asimétrico: Otros pueden cifrar mensajes que solo tú puedes descifrar
• PKI Infrastructure: Fundamento de toda infraestructura de clave pública

⚠️ Seguridad:
• NUNCA compartas tu clave privada — es como tu contraseña maestra
• La clave pública derivada SÍ puede compartirse libremente
• Guarda copias de respaldo seguras de tus claves privadas
• En producción, siempre encripta la clave privada con passphrase
• Si comprometes tu clave privada, revócala inmediatamente

⚡ Uso: Selecciona el tipo de algoritmo y tamaño de clave según tus necesidades de seguridad, opcionalmente protege con contraseña, y genera la clave. Después usa "Extract Public Key" para obtener la clave pública derivada.
⚙ Key type
openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048

▶ ℹ️ EXTRACT PUBLIC KEY - EXTRACCIÓN DE CLAVE PÚBLICA (click para ver ayuda)
Esta herramienta extrae la clave pública de una clave privada usando openssl pkey -pubout.

📝 Datos a Ingresar:
• Private Key (PEM): La clave privada en formato PEM (-----BEGIN PRIVATE KEY-----)
  • Puede ser RSA, EC (Elliptic Curve), o cualquier algoritmo asimétrico
  • Puede estar encriptada con passphrase o sin encriptar
  • Soporta formatos: PKCS#1, PKCS#8, SEC1

🔓 ¿Qué hace?
En criptografía asimétrica, cada par de claves tiene:
  • Clave Privada: Secreta, nunca se comparte, usada para firmar y descifrar
  • Clave Pública: Derivada matemáticamente de la privada, se comparte libremente, usada para verificar firmas y cifrar
Esta herramienta realiza la derivación matemática para extraer la parte pública de tu clave privada.

🔑 ¿Para qué se usa la clave pública?
• Distribución segura: Puedes compartir tu clave pública libremente sin comprometer seguridad
• Verificación de firmas: Otros pueden verificar que TÚ firmaste un documento/mensaje
• Cifrado hacia ti: Otros pueden cifrar mensajes que solo TÚ puedes descifrar con tu clave privada
• Certificados SSL/TLS: La clave pública se incluye en el CSR y certificado
• SSH authorized_keys: Añadir tu clave pública a servidores remotos para autenticación
• PKI/CA: Las CAs usan tu clave pública para firmar tu certificado

⚡ Uso: Pega tu clave privada (puede ser la que generaste arriba), haz clic en "Extract", y obtendrás la clave pública derivada que puedes compartir con otros.

▶ ℹ️ CERTIFICATE GENERATION - GENERACIÓN DE CERTIFICADOS, CADENAS Y FORMATOS (click para ver ayuda)
Esta herramienta te permite generar certificados digitales X.509, construir una cadena simple CA + Leaf, reutilizar una clave privada existente y trabajar con formatos PEM / PFX / PKCS#12.

🧩 MODOS DISPONIBLES:
• Nuevo Certificado: genera un certificado nuevo, ya sea Self-Signed o CSR
• From PFX: permite partir de material exportado o preparado para empaquetado en PKCS#12 / PFX
• From PEM: convierte material PEM hacia un contenedor PFX / PKCS#12 o extrae contenido desde un PFX según el flujo del módulo

📝 CONFIGURACIÓN PRINCIPAL:
• Type: define si generarás un certificado Self-Signed o un CSR
  • Self-Signed: genera un certificado autofirmado útil para laboratorios, desarrollo local y pruebas internas
  • CSR: genera una solicitud de certificado para enviarla a una CA externa o institucional
• Output Format: define el formato final de exportación
  • PEM (.pem / .crt): formato texto usado por OpenSSL, servidores web y herramientas de laboratorio
  • PFX (.pfx / .p12): contenedor usado comúnmente en Windows, navegadores e importación de certificados con clave privada

🪪 IDENTITY:
• Common Name (CN): nombre principal del certificado. Para TLS normalmente representa el dominio o hostname
• Organization (O): nombre de empresa, institución o laboratorio
• Organizational Unit (OU): departamento o área responsable
• Email: dato opcional de contacto administrativo

📍 LOCATION:
• Country (C): código ISO de 2 letras, por ejemplo HN, US, ES
• State / Province (ST): departamento, estado o provincia
• Locality / City (L): ciudad o localidad

🔐 CRYPTO SETTINGS:
• Key Size (RSA):
  • 2048 (Standard): valor recomendado por defecto
  • 3072 (High Security): seguridad extendida
  • 4096 (Maximum Security): máxima seguridad con mayor costo computacional
• Hash Function: algoritmo usado para la firma del certificado
  • SHA-256: recomendada por defecto
  • SHA-384 / SHA-512: opciones más fuertes según el entorno
• Certificate Validity (days): duración del certificado final o leaf certificate

📦 ARCHIVOS Y CLAVES:
• Base File Name: prefijo usado para nombres como certificate.key.pem, certificate.crt.pem o certificate.pfx
• Password (PFX): requerida cuando exportas en formato PFX / PKCS#12
• Existing Private Key (PEM): opcional; permite reutilizar una clave privada existente en lugar de generar una nueva

🏗️ GENERATE CHAIN (CA + LEAF):
Cuando activas esta opción, la herramienta genera una pequeña cadena de confianza:
  • CA Common Name: nombre de la autoridad certificadora local
  • CA Certificate Validity (days): duración del certificado CA, normalmente mayor que la del certificado final
  • Uso típico: laboratorios PKI, pruebas TLS internas, validación de cadenas y clases

📁 FROM PFX:
Este flujo te ayuda a trabajar con material relacionado con archivos PKCS#12 / PFX.
  • Output Base Name: prefijo para los archivos generados
  • Quick Load Files: detecta automáticamente archivos .pfx, .p12, .crt, .key y .pem
  • Certificate (PEM): certificado en formato PEM
  • Private Key (PEM): clave privada en formato PEM
  • CA / Intermediate Certificates: cadena opcional de intermedios o CA
  • Private Key Password: password de una clave privada PEM cifrada, solo si aplica
  • PFX File Password: password del contenedor PKCS#12 / PFX
  • OpenSSL Reference Commands: muestra los comandos de referencia usados para empaquetar o convertir material certificado/clave

📤 FROM PEM / EXTRACCIÓN:
Este flujo trabaja con extracción de material desde un archivo PFX:
  • PFX File: archivo fuente .pfx / .p12
  • PFX / PKCS#12 Password: password usado para abrir el contenedor
  • Exported Private Key Password: password opcional para cifrar la key.pem exportada
  • Output File Name (Key / Certificate / Chain): nombres sugeridos para los archivos exportados
  • Reference Commands: comandos OpenSSL educativos para extraer certificado, clave privada y cadena
  • Outputs: muestra por separado Private Key, Certificate y CA Chain

🔐 ¿PARA QUÉ SE USA?
• HTTPS / TLS en laboratorios y desarrollo
• Generación de certificados autofirmados y CSR
• Construcción de cadenas CA + Leaf para pruebas
• Conversión entre PEM y PFX / PKCS#12
• Exportación de material certificado/clave para Windows, OpenSSL o navegadores
• Ejercicios de PKI educativa y validación con OpenSSL

⚡ USO RECOMENDADO:
Para un escenario simple usa Self-Signed + PEM + RSA 2048 + SHA-256 + 365 days.
Si necesitas una cadena de pruebas, activa Generate Chain y define una validez mayor para la CA que para el certificado final.
Si vas a exportar a Windows o navegador, utiliza PFX / PKCS#12 con password.

▶ ℹ️ LIVE CERTIFICATE VALIDATION - VALIDACIÓN EN TIEMPO REAL (click para ver ayuda)
Esta herramienta te permite conectarte a un servidor real y extraer su certificado SSL/TLS en tiempo real usando openssl s_client.

📝 Datos a Ingresar:
• Hostname: El dominio o dirección IP del servidor (ej: www.google.com, github.com, 8.8.8.8)
• Port: El puerto donde escucha el servicio SSL/TLS (443 para HTTPS, 465 para SMTPS, 993 para IMAPS)
• Options: Parámetros adicionales de OpenSSL:
  • -showcerts: Muestra toda la cadena de certificados (recomendado)
  • -servername: Añade SNI (Server Name Indication) - necesario para servidores con múltiples dominios
  • -brief: Muestra solo información básica del certificado

🔍 ¿Por qué es útil?
• Inspección en vivo: Ver el certificado que un servidor está presentando actualmente
• Diagnóstico: Detectar problemas de configuración SSL/TLS (expired, self-signed, wrong chain)
• Auditoría de seguridad: Verificar qué algoritmos, versiones de TLS y cifrados usa el servidor
• Troubleshooting: Identificar problemas con SNI, cadenas incompletas o certificados intermedios faltantes
• Aprendizaje: Entender cómo funciona el handshake SSL/TLS y qué información intercambian cliente-servidor

⚡ Uso: Ingresa el hostname del servidor que deseas inspeccionar, ajusta el puerto si no es 443, y haz clic en "Validate Certificate" para conectarte y extraer el certificado.
openssl s_client -connect example.com:443 -showcerts
enter hostname and click Validate Certificate
▶ ℹ️ UNDERSTANDING CERTIFICATE VALIDATION - DOS TIPOS DE VALIDACIÓN (click para ver ayuda)
La validación de certificados SSL/TLS tiene DOS capas de seguridad separadas que los navegadores verifican:

1️⃣ Validación Criptográfica de Cadena (Chain Validation):
• Verifica que el certificado esté firmado correctamente por una CA confiable
• Valida toda la cadena: Certificado → CA Intermedia → CA Raíz
• Comprueba fechas de validez (notBefore / notAfter)
• Verifica firmas digitales y algoritmos
• Comando: openssl verify -CAfile chain.pem certificate.pem

2️⃣ Validación de Hostname/Dominio (Hostname Validation):
• Verifica que el dominio solicitado coincida con el certificado
• Compara contra el Common Name (CN) y Subject Alternative Names (SAN)
• Soporta wildcards (*.ejemplo.com)
• ⚠️ Nota: openssl verify NO valida hostname — es una verificación adicional

🔐 Ambas validaciones deben pasar para que HTTPS sea seguro. Esta herramienta permite verificar cada capa por separado para fines educativos.
Paste el certificado del servidor que quieres validar. Ejemplo: openssl s_client -connect example.com:443 -showcerts | openssl x509
Paste la cadena completa de CA (intermedio + raíz). Los certificados pueden estar uno después del otro.
Si se proporciona, se validará que el hostname coincida con CN o SAN del certificado
openssl verify -CAfile chain.pem certificate.pem
